### BxOS의 스케줄러 존재 여부 및 동작 방식

네, **BxOS(Haribote OS 기반)에는 스케줄러가 구현되어 있습니다.** 

BxOS는 시분할(Time-sharing) 방식의 멀티태스킹을 지원하며, 주요 로직은 `harib27f/haribote/mtask.c`와 `timer.c`에 구현되어 있습니다.

#### 1. 스케줄링 알고리즘
*   **다중 레벨 우선순위 큐 (Multi-level Priority Queue)**: 총 10개(`MAX_TASKLEVELS`)의 우선순위 레벨을 가집니다.
*   **레벨 간 스케줄링**: 숫자가 낮은 레벨(0번 레벨 등)에 실행 가능한 태스크가 있다면 해당 레벨을 먼저 처리합니다.
*   **레벨 내 스케줄링 (Round-Robin)**: 동일한 우선순위 레벨 내에 여러 태스크가 있을 경우, 순차적으로 한 번씩 실행 기회를 주는 라운드 로빈 방식을 사용합니다.

#### 2. 태스크 스위칭 메커니즘
*   **하드웨어 태스크 스위칭**: x86 아키텍처의 **TSS(Task State Segment)** 기능을 사용하여 레지스터 상태를 보존하고 복구합니다. 소스 코드 상에서는 `farjmp` 명령을 통해 다른 TSS 셀렉터로 점프함으로써 문맥 전환(Context Switch)을 수행합니다.
*   **타이머 기반 트리거**: PIT(Programmable Interval Timer)로부터 발생하는 **IRQ0 인터럽트**가 스케줄링의 원동력입니다. `timer.c`의 `inthandler20` 함수에서 현재 태스크의 할당 시간(Time slice)이 끝나면 `task_switch()`를 호출합니다.

#### 3. 주요 관련 함수 (`mtask.c`)
*   `task_init()`: 태스크 관리 시스템을 초기화하고 메인 태스크를 등록합니다.
*   `task_alloc()`: 새로운 태스크를 위한 메모리와 TSS를 할당합니다.
*   `task_run()`: 태스크를 특정 우선순위 레벨에 추가하고 실행 가능한 상태로 만듭니다.
*   `task_sleep()`: 태스크를 대기 상태로 전환하여 CPU를 사용하지 않도록 합니다.
*   `task_switch()`: 다음에 실행할 태스크를 결정하고 문맥 전환을 실행합니다.

따라서 BxOS는 사용자 애플리케이션과 시스템 태스크들이 동시에 동작할 수 있도록 관리하는 완전한 형태의 스케줄러를 갖추고 있습니다.

### BxOS 멀티태스킹 샘플 프로그램 및 테스트 방법

BxOS에는 멀티태스킹 기능을 효과적으로 보여주는 여러 샘플 프로그램이 포함되어 있으며, 간단한 콘솔 명령어를 통해 여러 태스크를 동시에 실행하는 테스트를 진행할 수 있습니다.

#### 1. 멀티태스킹 활용 샘플 프로그램
가장 대표적인 예제들은 다음과 같습니다:

*   **`noodle` (`harib27f/noodle/noodle.c`)**: 1초마다 숫자가 올라가는 타이머 프로그램입니다. 여러 개를 띄워 놓으면 각각 독립적으로 시간이 흐르는 것을 볼 수 있습니다.
*   **`lines` (`harib27f/lines/lines.c`)**: 화면에 선을 반복해서 그리는 프로그램입니다. CPU를 지속적으로 사용하므로 스케줄링 동작을 확인하기 좋습니다.
*   **`stars2` (`harib27f/stars2/stars2.c`)**: 창 안에 무작위로 별을 그립니다.
*   **`walk` (`harib27f/walk/walk.c`)**: 캐릭터가 걸어 다니는 애니메이션을 보여줍니다.

#### 2. 여러 태스크 동시 실행 테스트 방법
BxOS 콘솔(A.HE2 등)에서 `start` 명령어를 사용하면 새 창을 열면서 프로그램을 백그라운드 태스크로 실행할 수 있습니다.

**테스트 시나리오:**
1.  BxOS 부팅 후 콘솔 창을 엽니다.
2.  콘솔에 `start noodle`을 입력합니다. (새 창에서 타이머가 시작됩니다.)
3.  다시 원래 콘솔로 돌아와 `start lines`를 입력합니다. (또 다른 창에서 선 그리기가 시작됩니다.)
4.  필요에 따라 `start walk` 등을 추가로 입력합니다.
5.  이제 여러 개의 윈도우가 동시에 움직이는 것을 확인할 수 있습니다. 이것이 BxOS 스케줄러에 의해 여러 태스크가 시분할로 실행되고 있는 상태입니다.

#### 3. 실행 상태 확인 및 모니터링
현재 얼마나 많은 태스크가 실행 중인지 확인하려면 다음 도구들을 사용하세요.

*   **`task` 명령어**: 콘솔에서 `task`를 입력하면 현재 실행 중인 태스크의 ID, 이름, 우선순위 레벨, CPU 점유 시간 등을 텍스트로 출력합니다.
*   **`taskmgr` (태스크 매니저)**: 콘솔에서 `taskmgr`을 입력하면 GUI 기반의 작업 관리자가 실행됩니다. 각 태스크의 리스트와 CPU 사용률 그래프를 실시간으로 확인할 수 있어 멀티태스킹 상태를 시각적으로 파악하기에 가장 좋습니다.

이러한 방법들을 통해 BxOS의 스케줄러가 여러 프로그램을 어떻게 동시에 관리하고 실행하는지 직접 테스트해 보실 수 있습니다.
